## What are Workflows?

## Workflow Execution

### Start a Workflow

```python
import requests

response = requests.post(
    f"https://catalogapi.rastro.ai/api/public/workflows/{workflow_id}/execute",
    headers={"Authorization": f"Bearer {api_key}"},
    json={
        "input": [
            {
                "sku": "LAPTOP-001",
                "title": "Professional Laptop",
                "raw_specs": "Intel i7, 16GB RAM, 512GB SSD"
            }
        ]
    }
)

run = response.json()
print(f"Started: {run['workflow_run_id']}")
```

### Poll for Status

```python
import time

while True:
    status = requests.get(
        f"https://catalogapi.rastro.ai/api/public/workflows/runs/{run_id}",
        headers=headers
    ).json()

    print(f"Progress: {status['progress']*100:.0f}%")

    if status["status"] in ["completed", "failed"]:
        break

    time.sleep(5)
```

### Get Results

```python
if status["status"] == "completed":
    results = status["results"]["data"]
    for item in results:
        print(f"SKU: {item['sku']}")
        print(f"AI Description: {item['ai_description']}")
```

## Execution Status

| Status | Description |
|--------|-------------|
| `pending` | Queued for execution |
| `running` | Currently processing |
| `completed` | Successfully finished |
| `failed` | Error occurred |
| `cancelled` | Manually stopped |

## Input Requirements

- Workflows must have **exactly one input node**
- Input data is an **array of objects**
- Each object represents one record to process

```json
{
  "input": [
    {"sku": "A", "title": "Product A"},
    {"sku": "B", "title": "Product B"},
    {"sku": "C", "title": "Product C"}
  ]
}
```

## Output Structure

Completed workflows return:

```json
{
  "workflow_run_id": "run_abc123",
  "status": "completed",
  "progress": 1.0,
  "results": {
    "data": [
      {
        "sku": "A",
        "title": "Product A",
        "ai_description": "Generated description...",
        "ai_category": "Electronics"

    "total": 3,
    "page": 1,

  "node_results": {
    "node_uuid": {
      "node_type": "attribute_mapper",

      "total": 3

  }
}
```

## Error Handling

```python
if status["status"] == "failed":
    print(f"Error: {status['message']}")

    # Check node-level errors
    for node_id, node_result in status.get("node_results", {}).items():
        if "error" in node_result:
            print(f"Node {node_id} failed: {node_result['error']}")
```

## Best Practices

<AccordionGroup>
  <Accordion title="Batch Your Data">
    Send multiple records per execution rather than one at a time
  </Accordion>

  <Accordion title="Poll Efficiently">
    Use exponential backoff when polling (3s → 5s → 10s → 15s)
  </Accordion>

  <Accordion title="Handle Failures Gracefully">
    Implement retry logic for transient failures
  </Accordion>

  <Accordion title="Save Results">
    Store workflow results in your catalog or database immediately after completion
  </Accordion>
</AccordionGroup>
